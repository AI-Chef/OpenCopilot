<Warning>
This endpoint is protected by the same JWT that the user uses to call other APIs. The response will include data for the specified chatbot, along with other metadata information.
</Warning>

# Retrieve Conversation Data

This endpoint allows you to retrieve the conversation data for a specific conversation ID. The response includes the message history, sentiment, and reason for handoff (if applicable).

The endpoint is paginated, and you can specify the `offset` and `limit` parameters to control the amount of data returned.

<RequestExample>
```bash Example Request
curl --location 'http://localhost:8888/backend/datalake/conversations/15fd3fb5-cb2f-4d41-ba00-5dc79ad8273d' \
  --header 'Authorization: Bearer TOKEN'
```
</RequestExample>

<ResponseExample>

```json Response
        "debug_json": null,

        "from_user": false,

        "knowledgebase_called": false,
        "message": "Hi"
      },
      {
        "api_called": false,
        "created_at": null,
        "debug_json": null,
        "extra_params": null,
        "from_user": true,
        "id": 63,
        "knowledgebase_called": false,
        "message": "Hello"
      }
    ],
    "session_id": "session1",
    "summary": "First Session"
  }

```
</ResponseExample>

The response includes the following fields:

- `messages`: An array of message objects, each containing information about the message, such as whether it was an API call, whether the knowledge base was called, the message text, and more.
- `session_id`: The unique identifier for the conversation session.
- `summary`: A brief summary of the conversation.

You can use the `offset` and `limit` parameters to control the pagination of the response. For example, to retrieve the first 10 messages, you can use the following request:

<RequestExample>
```bash Example Request
curl --location 'http://localhost:8888/backend/datalake/conversations/15fd3fb5-cb2f-4d41-ba00-5dc79ad8273d?offset=0&limit=10' \
  --header 'Authorization: Bearer TOKEN'
```
</RequestExample>
